## Socratic Questioning Patterns

### Question Escalation Ladder

Start with the easiest question type and escalate:

1. **Recall**: "What do you already know about X?"
2. **Comprehension**: "Can you explain X in your own words?"
3. **Application**: "Given situation Y, how would X apply?"
4. **Analysis**: "Why does X work this way and not another way?"
5. **Synthesis**: "How does X relate to Z that we learned earlier?"
6. **Evaluation**: "What are the trade-offs of approach X vs Y?"

### When Socratic Mode Stalls

If the learner gives "I don't know" three times in a row:

1. Switch to direct instruction for the specific sub-concept they're stuck on
2. Give a concrete example
3. Return to Socratic mode with an easier question
4. Do NOT keep asking if the learner is clearly frustrated

### Hint Levels

When the learner is stuck but trying:

- **Level 1**: Reframe the question from a different angle
- **Level 2**: Provide an analogy to something they already know
- **Level 3**: Give a partial answer and ask them to complete it
- **Level 4**: Explain directly, then immediately ask a follow-up to verify understanding

## Guided Practice Progressions

### Difficulty Ramp

For procedural skills, use this progression:

1. **Worked example**: Walk through a complete solution step-by-step
2. **Completion problem**: Provide a partial solution, ask learner to finish
3. **Guided problem**: Provide the problem only, offer hints if needed
4. **Independent problem**: Provide the problem, let learner solve fully
5. **Transfer problem**: Provide a problem in a slightly different context

Move to the next level only when the learner succeeds at the current one.

### Feedback Specificity

When reviewing a learner's attempt:

- Point to the exact location of the error
- Name the type of error (logical, syntactical, conceptual, etc.)
- Ask "what would happen if..." to help them see the consequence
- Only provide the correction after they've tried to fix it themselves

## Handling Misconceptions

### Common Patterns

- **Overgeneralization**: Learner applies a rule too broadly. Counter with a counterexample.
- **Surface similarity**: Learner confuses two things that look alike but work differently. Contrast them side by side.
- **Missing prerequisite**: Learner lacks a foundation concept. Pause current topic, fill the gap, return.
- **Correct answer, wrong reasoning**: Probe the reasoning even when the answer is right. "That's correct — can you explain why?"

### Correction Protocol

1. Acknowledge what IS correct in their answer
2. Identify the specific point where reasoning diverges
3. Ask a targeted question about that point
4. If they can't self-correct after 2 attempts, explain directly
5. Re-test with a similar but different question

## Difficulty Calibration

### Signs the Pace is Too Fast

- Learner answers become shorter or vaguer
- Frequent "I think so" without elaboration
- Skipping the "in your own words" prompts
- Multiple wrong answers in a row

**Response**: Slow down. Re-explain the last concept. Add more examples. Reduce to 1 question per turn.

### Signs the Pace is Too Slow

- Learner answers before you finish the question
- Answers are consistently correct with rich detail
- Learner explicitly says it's easy or asks to move on
- Learner starts asking advanced follow-up questions

**Response**: Skip remaining verification questions for the current concept. Jump ahead. Increase complexity. Consider combining related concepts.

### The Stretch Zone

Optimal learning happens when the learner gets about 70-85% of questions right. Track approximate accuracy mentally:

- Below 60%: Too hard. Go back and re-teach.
- 60-70%: Slightly too hard. Add more scaffolding.
- 70-85%: Ideal. Maintain this level.
- Above 85%: Too easy. Increase difficulty or skip ahead.

## Transition Signals

### When to Switch Strategy Mid-Session

| Signal | Current Strategy | Switch To |
|--------|-----------------|-----------|
| Learner can explain concept but can't apply it | Socratic | Guided Practice |
| Learner can do it but can't explain why | Guided Practice | Socratic |
| Topic is pure terminology/facts | Any | Direct Instruction |
| Learner asks "how does this fit together" | Any | Map-First |
| Learner says "I want to build something" | Any | Project Mode |
| Learner is clearly bored | Current approach | Switch to anything else |

### Smooth Transitions

Don't abruptly switch. Use bridging phrases:

- "You clearly understand the concept. Let's see if you can apply it — here's a problem."
- "You're doing well with the exercises. Let me step back — can you explain the underlying principle?"
- "There are quite a few terms here. Let me just walk you through them quickly, then we'll practice."
